Memory Safety

no invalid memory accesses

  • no buffer overflow
  • no dangling pointers
  • no data races
  • no use after free
  • no double free
  • no null de-referencing

Ownership & Lifetimes

Memory safety guaranteed at compile time

Ownership Rules:

  • Each value has an "owner"
  • There can only be one owner
  • When the owner goes out of scope, the value will be dropped

Borrowing:

  • Multiple immutable references are allowed
  • A single mutable reference
  • A borrowed value always has to live longer than its references

Ownership and lifetimes allow the compiler to statically check the usage of resources; including memory
